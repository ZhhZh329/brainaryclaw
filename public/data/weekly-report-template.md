# 周报模板（新版）

## 1. 本周核心任务
请只写 **1–3 个最重要的主线任务**。不要写流水账，不重要的事务压缩。

对每个任务，写清楚：

- **本周做了什么**
- **本周新增了什么硬进展**
  - 新实验
  - 新数据
  - 新模块
  - 新 benchmark
  - 新失败模式
  - 新方法资产
- **当前卡点是什么**
- **下周最小可验证动作是什么**

---

## 2. CoT 增量
这里不是简单重复“做了什么”，而是写：

- 这周通过具体工作，得到了哪些**直接认知更新**
- 哪些判断比上周更清楚了
- 哪些方法、路径、实验设计被证实有效 / 无效

---

## 3. CoT’s CoT（最重要）
这里写的不是任务本身，而是**对思考方式、研究方法、系统结构的更高层理解**。

请严格按下面格式写：

### CoT’s CoT 1
- **内容：**
- **这是老师原本不知道的吗：是 / 否**
- **如果“是”**：
  - 老师为什么原本不知道？
  - 信息差在哪里？
  - 这次新增认知是什么？
- **如果“否”**：
  - 为什么这件事还没有被自动化科研吸收？
  - 为什么还没有变成 workflow / agent / benchmark / module / memory？
- **自动化潜力：高 / 中 / 低**
- **建议沉淀形式：**
  - prompt
  - workflow
  - benchmark
  - memory schema
  - agent module
  - evaluation protocol
  - 其他

### CoT’s CoT 2
同上

> 注意：
> 如果这里写的内容老师早就知道，那你就必须继续回答：
> **既然老师知道，为什么还没有自动化？**
> 如果回答不了，说明这条 CoT’s CoT 还不够深。

---

## 4. 和 Brainary 的关联
不要只写“和 Brainary 有关”或者“对 Brainary 有启发”。

必须写成**能力映射**：

- 这项工作对应 Brainary 的什么能力？
  - memory
  - reasoning
  - calibration
  - cognitive control
  - tool orchestration
  - risk modeling
  - workflow management
  - benchmark / evaluation
  - 其他
- **这是老师原本不知道的吗：是 / 否**
- 如果“是”：
  - 老师为什么原本不知道？
  - 这项关联的新意在哪里？
- 如果“否”：
  - 为什么这项能力还没有被做成系统？
- 最终可能沉淀成什么：
  - 模块
  - agent
  - benchmark
  - workflow
  - memory 结构
  - 评测协议

---

## 5. 对 meta-cognition 的新理解
这里不要写空泛感想，要写**新的元认知理解**。

格式要求：

- **新的理解是什么**
- **这是老师原本不知道的吗：是 / 否**
- 如果“是”：
  - 为什么老师原本不知道？
  - 这次理解新增在哪里？
- 如果“否”：
  - 为什么这个理解还没有进入自动化科研体系？
- 这个理解未来能否变成：
  - 明确规则
  - 调度策略
  - 监控机制
  - agent policy
  - cognitive control module

---

## 6. 题目是否值得做（必须单独判断）
这一部分必须单独回答，不允许默认“值得做”。

请写清楚：

- **这个题目解决的真实问题是什么？**
- **谁会为这个能力买单？**
- **它是局部优化，还是通用能力？**
- **如果做成，最可能沉淀成什么？**
  - 论文
  - benchmark
  - workflow
  - module
  - product feature
  - 其他
- **如果现在还看不出价值，为什么还继续做？**

---

## 7. 价值观总结（必须写金额，不是分数，不是高低）
这里的“价值”指的是**金额**，不是“重要 / 不重要”。

对每个主要任务，写：

- **任务名称**
- **估计价值金额**
- **计算方法**
- **参考依据**

可参考依据包括：

- 对应软件 / 产品定价
- 相关公司的估值
- 市场规模
- 可替代人工成本
- 竞品能力
- AI 辅助给出的合理定价逻辑

格式示例：

### 任务 A
- **估计价值：** 50 万人民币
- **计算方法：**
- **参考依据：**
- **为什么值这个钱：**

> 注意：
> 不要再写“85 分 / 90 分 / 高优 / 高价值赛道”这种话。
> 这些都不算价值计算。

---

## 8. CoT's CoT 证据等级
每一条 CoT's CoT 额外标注一个证据等级：

- **A：有实验 / 数据直接支撑**
- **B：有案例 / 现象支撑，但未系统验证**
- **C：当前只是推测 / 方向判断**

目的是区分：
- 哪些是真正被验证的认知增量
- 哪些还只是 proposal
- 哪些只是灵感

---

## 9. 项目管理（必须写）
每个主线项目都要补充项目管理信息，不能只写研究内容，不写推进方式。

请至少写清楚：

- **项目名称**
- **当前阶段**（想法 / survey / 原型 / 小规模实验 / 大规模实验 / 写作 / 投稿）
- **时间规划**
  - 原计划这周做到哪里
  - 实际这周做到哪里
  - 本周实际投入时间
  - 下周预计投入时间
  - 关键里程碑时间点
- **资源规划**
  - 需要的人（是否缺人，谁负责）
  - 需要的算力 / 数据 / API / GPU / 沙盒 / 工具
  - 需要的外部支持（合作方、会议、助理协调等）
- **依赖关系 / 关键路径**
  - 我依赖谁的结果？
  - 谁依赖我的结果？
  - 如果我延期，会卡住谁？
  - 当前这个项目是不是关键路径上的项目？是 / 否
- **Token / 成本规划**
  - 本周大致 token / API 消耗
  - 当前是否存在 token / quota / 费用瓶颈
  - 是否有更省 token 的替代方案
- **进度判断**
  - 当前进度是否符合预期：是 / 否
  - 如果不符合预期，偏差在哪里？
  - 原计划 vs 实际完成 的主要差距是什么？
  - 偏差原因是：认知问题 / 方法问题 / 资源问题 / 执行问题 / 协作问题 / 其他
- **纠偏动作**
  - 下周准备怎么纠偏
  - 是否需要老师介入

项目管理的核心不是写形式，而是回答：
- 这个项目是不是按预期在推进？
- 如果没有，究竟卡在认知、资源、执行还是协作？
- 现在最需要调整的是题目、方法、资源还是人？

---

## 10. 自动化科研转化路径
请回答这项内容最适合如何进入自动化科研：

- **最适合先转成什么？**
  - prompt
  - checklist
  - workflow
  - harness
  - benchmark
  - memory schema
  - agent module
  - evaluator
  - 其他
- **当前转化门槛是什么？**
- **现在卡在哪一步？**
- **预计谁来做？**

---

## 11. 我这周最希望老师学到的新东西
这一部分必须单独写一句：

- **如果老师这周只能从我的周报里学到一件新东西，那应该是什么？**

这句话必须具体，不能空泛。

---

## 12. 申请 / 签字附件规则
以后任何**申请**和**签字**之前，必须把与该事项直接相关的周报记录 **attach** 上，一并提交给老师。

最低要求：
- 先在周报里写清相关事项的背景、进展、当前卡点、下一步动作
- 再发起申请 / 签字
- 申请材料里必须附上对应周报记录或周报截图 / 链接
- 如果周报里没有记录，视为材料不完整，不进入申请 / 签字流程

---

## 13. 提交后补全规则
提交后如果缺少以下任一内容，需要**立即补全**：

- 缺少 CoT’s CoT
- 没有标注“老师原本不知道的吗：是 / 否”
- 没有解释“为什么老师不知道 / 为什么还没有自动化”
- 没有写和 Brainary 的能力映射
- 没有写对 meta-cognition 的新理解
- 没有单独判断“题目是否值得做”
- 价值观总结没有写金额和计算方法
- 没有给 CoT's CoT 标证据等级
- 没有写项目管理（时间 / 资源 / token / 进度偏差 / 纠偏动作）
- 没有写自动化科研转化路径
- 仍然是流水账
- 没有写“如果老师这周只能从我的周报里学到一件新东西，那应该是什么？”

---

## 13. 评分机制（老师内部评分参考）
每次周报按以下维度评分，总分 110：

### 1) 硬进展分（20）
- 0–5：基本无硬进展
- 6–10：有少量推进，但不清晰
- 11–15：有明确实验 / 数据 / 模块 / benchmark 进展
- 16–20：硬进展扎实，且和主线高度相关

### 2) 思考 delta 分（20）
- 0–5：基本是做事记录
- 6–10：有一些判断，但较浅
- 11–15：有明确认知更新
- 16–20：有让老师学到新东西的增量

### 3) CoT’s CoT 质量分（20）
- 0–5：没有 CoT’s CoT，或很空
- 6–10：有 meta-level 表述，但不够深
- 11–15：有较明确方法论或系统层判断
- 16–20：CoT’s CoT 清晰、深刻、可转化

### 4) Brainary 关联分（15）
- 0–3：没写 / 很空
- 4–7：有方向，但泛
- 8–11：能落到具体能力
- 12–15：能力映射明确，且有沉淀路径

### 5) 价值判断分（15）
- 0–3：没有价值判断
- 4–7：只有重要性 / 优先级
- 8–11：有价值逻辑，但金额不够扎实
- 12–15：价值、金额、计算方法都清楚

### 6) 自动化科研转化分（10）
- 0–2：完全没写
- 3–5：写了但很泛
- 6–8：有明确转化路径
- 9–10：转化目标、门槛、责任人都清楚

### 7) 项目管理分（10）
- 0–2：完全没有项目管理信息
- 3–5：写了时间 / 资源 / token 之一，但不完整
- 6–8：时间、资源、token、进度判断基本齐全
- 9–10：能明确写出进度偏差、原因和纠偏动作

### 红线扣分项
- 流水账：-15
- 没标“老师原本不知道的吗”：-10
- 没解释“为什么老师不知道 / 为什么还没自动化”：-10
- 价值只打分不写金额：-10
- 没写“如果老师这周只能学到一件事”：-5
- CoT’s CoT 没有证据等级：-5

### 最终等级
- A（94–110）：高质量，值得重点跟进
- B（77–93）：合格，有东西，但还需补强
- C（55–76）：偏弱，主要是流水账或思考不够深
- D（0–54）：不合格，需重写 / 补交

### 老师动作建议（必选其一）
- KEEP：继续推进
- REFINE：继续做，但要补思考 / 价值
- MEET：需要约会讨论
- REWRITE：周报重写
- RESCOPE：题目要收缩或换方向
- STOP：题目价值不足，建议停掉

---

## 14. 禁止事项
以下写法以后不合格：

- 流水账
- 只列任务，不写认知增量
- 只写 CoT，不写 CoT’s CoT
- 不标“老师已知 / 未知”
- 不解释“为什么老师不知道 / 为什么还没自动化”
- 只写“和 Brainary 有关”，不做能力映射
- 不判断“题目值不值得做”
- 价值只打分，不写金额和计算方法
- 不写项目管理（时间 / 资源 / token / 进度偏差 / 纠偏动作）
- 不写自动化科研转化路径
